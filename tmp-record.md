这是一个非常经典且初学者极易踩坑的 Canvas 问题！简单来说：CSS 设置的是“显示尺寸”（Display Size），而 HTML 属性设置的是“画布分辨率”（Drawing Buffer Size）。
当你只通过 CSS 设置宽高，而不设置 HTML 属性时，Canvas 的默认分辨率（300x150）与 CSS 的显示尺寸（322x180）不一致，浏览器就会强行拉伸画布，导致绘制的内容模糊或变形。

核心区别
HTML 属性 (canvas.width / canvas.height)：决定了画布内部有多少个“像素点”（即分辨率）。如果不设置，Canvas 的默认宽度是 300px，高度是 150px。这相当于你买了一张固定尺寸的画布。
CSS 样式 (style.width / style.height)：决定了这张画布在网页上被拉伸或缩小到多大。这相当于你把这张画布强行塞进一个相框里。
. 为什么效果不一样？
在你的代码中，.canvas-panel 的 CSS 尺寸是 322px × 180px。
如果你在 JS 中没有写 canvas1.width=322; canvas1.height=180;，那么画布的实际分辨率依然是默认的 300px × 150px。
此时，浏览器为了把 300×150 的画布塞进 322×180 的 CSS 框里，会进行非等比例的拉伸。这会导致两个后果：
图形变形：你画的正圆形（奥运五环的环）会被拉成椭圆。
画面模糊：因为像素被强行放大，边缘会出现锯齿或模糊。
. 正确的最佳实践
在绘制 Canvas 时，永远不要仅依赖 CSS 来设置宽高。正确的做法是：让 HTML 属性（分辨率）和 CSS 尺寸（显示大小）保持一致。

2. 内容丢失：重设宽高会清空画布

核心知识点总结（初学者必看）：
Canvas 绘图三步曲：每次画一个独立的图形，都必须遵循 beginPath() -> 画形状(如arc) -> stroke() 的顺序。
为什么每次都要 beginPath()？ 因为 Canvas 没有“图层”概念。如果不重新开始路径，arc() 只是把新的圆追加到旧的路径上，最后 stroke() 时，所有的圆会被当成一整条线画出来，导致颜色互相覆盖、线条连在一起。

关于奥运五环图案的间距规定，需要从“官方规范”和“实际绘图参考”两个层面来理解：

官方规范层面
根据《奥林匹克宪章》（Olympic Charter）的规定，奥运五环由五个尺寸相等（equal dimensions）的圆环组成，自左向右两两相互紧密交叉（interlaced）。

在官方规范中，并没有规定一个绝对固定的物理间距数值（如多少毫米），而是强调五个环的相对比例和交错关系。历史上，国际奥委会（IOC）在1986年曾规定过带有缝隙的版本，但在2010年，官方批准恢复了顾拜旦初版的“无缝（seamlessly）”交错设计。因此，最核心的规定是保持五个环大小一致，并呈现出标准的上下交错覆盖关系。

实际绘图参考层面
在实际的 CAD 或 CSS 绘图设计中，为了精准还原五环的“交叠感”与比例，通常会采用以下相对尺寸和间距参数作为参考：

相对尺寸比例：设定内圈半径为 1，外圈半径为 1.2。相邻圆环的圆心水平距离为 2.6，两排圆环的圆心垂直距离为 1.1。
绝对尺寸参考（以环宽为5为例）：若设定外环直径为50，内环直径为40（即环宽为5），那么横向间距通常设为 54，纵向偏移设为 22，重叠区域为 27。
CSS 布局参考（以单环100px为例）：若单环宽度设为 100px，环间水平间距设为 20px，则环中心距为 120px。上排三个环的 left 值依次为 0、120px、240px；下排两个环的 top 值为 60px，left 值分别为 60px 和 180px。

总结来说，官方规定的是五环“等尺寸、紧密交错”的几何逻辑；而在具体设计落地时，设计师会根据设定的基准尺寸，按照上述的比例参数（如水平间距、垂直偏移）来精确计算并绘制出标准的奥运五环。
